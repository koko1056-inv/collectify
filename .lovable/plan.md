

## マイルームの新コンセプト: 「推しと過ごすお部屋」

### コンセプト

「配置を頑張る場所」から「**毎日帰りたくなる、推しが住む小さな世界**」へ。  
どうぶつの森のような俯瞰視点の小部屋に、自分のアバターと推しグッズが自動的に並び、時間とともに変化する“居場所”を作ります。

### 体験の中心 (3つの柱)

1. **おまかせレイアウト** — グッズを「飾る」ボタン1つで自動配置。配置作業ゼロ。
2. **時間で変化する** — 朝/昼/夕/夜で部屋の光と色が変わる。アバターが部屋の中で動く。
3. **小さな相互作用** — 訪問者が「なでる」「いいね」を残す。お礼アイテムが部屋に増える。

### 削るもの (UIをすっきりさせる)

- 自由ドラッグ配置、回転、レイヤー操作、3D視点回転、Display Style選択 → **全部廃止**
- フルスクリーン編集モード → **廃止** (常に同じビューで完結)
- 編集モードと閲覧モードの区別 → **統合**

### 新UI構成 (`/my-room` の room タブ)

```text
┌──────────────────────────────────┐
│  ☀ おはよう  · 訪問 12 · ♥ 34   │ ← 上部ステータスpill (1行)
├──────────────────────────────────┤
│                                  │
│      [アイソメ2.5D 部屋]          │
│   ┌──────────────┐              │
│   │  🪴 棚にグッズ  │              │
│   │  🧸 床にぬいぐる │              │
│   │  👤 アバター     │              │
│   └──────────────┘              │
│                                  │
├──────────────────────────────────┤
│ [ + グッズを飾る ] [ 🎨 模様替え ] │ ← 2ボタンだけ
└──────────────────────────────────┘
```

- **グッズを飾る**: コレクションから選ぶ → 自動で空いてる場所に配置 (棚→床→壁の優先順)
- **模様替え**: テーマ(壁紙+床+光)を5〜8種類から選ぶだけ。家具個別配置なし

### 自動配置ロジック

部屋を「**スロット**」の集合として定義 (棚3段×4列、床4マス、壁2マスなど合計~20スロット)。  
グッズを追加すると空きスロット先頭に自動で挿入。グッズの種類(フィギュア/ぬい/缶バッジ)で適切なスロットを優先 (例: 缶バッジ→壁、ぬい→床)。  
ユーザーは「シャッフル」ボタンで再配置できる程度の操作のみ。

### 技術設計 (簡潔版)

- 既存の `binder_pages` (is_main_room) と `binder_items` テーブルは継続利用。`position_x/y` はスロット番号として再解釈。
- 新規: `room_themes` 定数 (壁紙×床×ライティングのプリセット)。`background_color` カラムに `theme:<id>` 形式で保存 (既存仕様継続)。
- レンダリング: SVG ベースのアイソメ2.5D (Three.js 不要 → 軽量化)。`IsometricRoomScene.tsx` を新規作成。
- 時間帯の光: CSS フィルター (`hue-rotate`, `brightness`) を時刻で切替。
- 既存 `Room3DEditor.tsx` (1177行) は **削除予定**。`ShelfView`, `FurnitureItem3D`, `DisplayStylePicker` 等の補助コンポーネントも未使用なら削除。

### 実装ステップ

1. **新コンポーネント** `IsometricRoomScene.tsx` を作成: スロット定義、アイテム自動配置、時間帯ライティング
2. **テーマプリセット** `roomThemes.ts` を整理: 5〜8種類 (ナチュラル/夜空/海辺/和室/カフェ etc.)
3. **新エディタ** `MyRoomScene.tsx` を作成: 上部ステータス + 部屋表示 + 下部2ボタン
4. **モーダル**: 「グッズを飾る」シート (コレクションから選択して `binder_items` に挿入)、「模様替え」シート (テーマ選択)
5. **MyRoomHome.tsx** の room タブを新コンポーネントに差し替え
6. **旧コード削除**: `Room3DEditor.tsx`、関連未使用ファイル
7. **モバイル最適化**: 390px 幅で部屋が綺麗に収まるよう調整

### 期待される効果

- UI要素が **15+ → 3つ** に減って迷わない
- 「配置作業」のストレスがゼロになり、毎日開きたくなる
- 時間帯変化と訪問者リアクションで「生きてる感」が出る
- コードベースも約 1500行 削減できる
